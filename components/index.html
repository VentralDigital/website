Nextra Components

NextJS

Links

All relative Markdown links are automatically converted to Next.js links. This means that the target page will be prefetched. And when you click on a link, the page will be loaded on the client-side like a SPA, without making a full page load. For example:

Click [here](/about) to read more.

Will be equivalent to:

import Link from 'next/link'
 
Click <Link href="/about">here</Link> to read more.

This feature makes navigation between Nextra pages fast and seamless.

Static Images

💡

This feature is enabled via staticImage: true in the Nextra config by default.

Nextra supports automatically optimizing your static image imports with the Markdown syntax. You do not need to specify the width and height of the image, just use the ![]() Markdown syntax:

index.mdx
![Hello](/og.png)

This loads the og.png file inside the public folder, and automatically wraps it with Next.js <Image>.

💡

You can also use ![](../public/og.png) to load the image from a relative path, if you don't want to host it via public.

The standard way to use Next.js Image (opens in a new tab) inside MDX is to directly import the component:

import Image from 'next/image'
 
<Image src="/og.png" alt="Hello" width={500} height={500} />

    LaTeX

    Using LaTeX within MDX is as simple as wrapping your expression in $ or $$.

    page.mdx
    The **Pythagorean equation**: $a=\sqrt{b^2 + c^2}$.
 
$$
a=\sqrt{b^2 + c^2}
$$

    will be rendered as:

    The Pythagorean equation: a=b2+c2a=\sqrt{b^2 + c^2}a=b2+c2​.

    a=b2+c2a=\sqrt{b^2 + c^2}a=b2+c2​

    You can still use Markdown and MDX syntax in the same line as your LaTeX expression.

    💡

    If you want to display $ in your content instead of rendering it as an equation, you can escape it with a backslash (\). For example \$e = mc^2\$ will be rendered as $e = mc^2$.

    To learn more about KaTeX and its supported functions and conventions, visit KaTeX’s documentation (opens in a new tab).

    Layout Bleeding

    When wrapping your content with <Bleed>, it will be slightly wider than the container and will overflow on both sides.

    There is nothing to writing. All you do is sit down at a typewriter and bleed.

    — Ernest Hemingway

    It provides a better reading experience when you want to present some graphical information, which normally looks nicer in a larger size.

    For example, you can put text, image, video or any component inside.

    You can even make it full-bleed using <Bleed full> to let it break out of the container completely.

    import { Bleed } from 'nextra-theme-docs'
 
<Bleed>Hey, I can use **Markdown** syntax here.</Bleed>
 
<Bleed full>
  ![Landscape](https://source.unsplash.com/eaxwP9J_V6s/1600x398)
</Bleed>
 
<Bleed full>
  <iframe
    src="https://codesandbox.io/embed/swr-states-4une7"
    width="100%"
    height="500px"
    title="SWR-States"
  />
</Bleed>
